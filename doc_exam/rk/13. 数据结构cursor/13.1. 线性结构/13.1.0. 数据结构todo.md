## 循序渐进学习路径

1. **打基础**：先精读 13.1.1. 线性结构知识点概述.md，弄清什么是线性关系、顺序/链式存储及线性表、栈、队列、串的区别与应用。读完后用自己的话总结一份“结构特征 + 场景”清单，帮助记忆。
2. **分模块突破**：
    - 13.1.2. 顺序表与链表.md / _java.md：对比顺序表和链表的存储结构、插删复杂度，再照着 Java 版实现跑一遍，体会指针操作与数组操作的差异。
    - 13.1.3. 队列与栈.md / _java.md：着重掌握“受限操作”思想，手写顺序栈/链栈、循环队列、双端队列。
    - 13.1.4. 串.md / _java.md：串其实就是字符线性表，关注模式匹配（如 KMP）等算法思想。
3. **配套练习**：每学完一个结构，至少做三件事：
    - 在草稿纸或白板画出插入/删除/查找流程；
    - 亲自写出核心操作代码，尤其是链式结构的指针处理；
    - 分析时间、空间复杂度，标注在哪些场景优势明显。
    - 可搭配 LeetCode 初级题（如 206. Reverse Linked List, 20. Valid Parentheses, 232. Implement Queue using Stacks）加深理解。
4. **项目化应用**：从实际问题出发选择结构：例如通讯录（线性表）、浏览器前进后退（栈）、打印缓冲区（队列）、文本编辑器（串）。尝试把其中一个场景实现成小程序，把数据结构嵌进去。
5. **复盘与扩展**：
    - 每周整理一次“知识脑图 + 代码仓库”，总结易错点。
    - 对比线性结构与树/图等非线性结构，思考什么时候需要升级数据结构。
    - 回顾 _java.md 文件中的实现细节，尝试加入单元测试或性能测量，提升工程化能力。

- 坚持“理解原理 + 动手实现 + 实战练习 + 定期复盘”四步走，就能稳固掌握线性结构，为后续学习复杂数据结构打好基础。
