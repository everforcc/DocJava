# Cursor AI模型使用指南

## 一、模型概览

### 1. Claude 3 Opus
- **定位**：最强大的大语言模型
- **特点**：
  - 超强的推理和分析能力
  - 最佳的上下文理解能力
  - 最高质量的代码生成
  - 最强的文档生成能力
- **适用场景**：
  - 复杂系统设计
  - 架构优化方案
  - 算法设计与实现
  - 长文档生成与组织
  - 深度技术研究

### 2. Claude 3 Sonnet
- **定位**：平衡型模型
- **特点**：
  - 性能与效率的良好平衡
  - 稳定的代码质量
  - 快速的响应速度
  - 准确的问题诊断
- **适用场景**：
  - 日常代码开发
  - 代码审查和重构
  - 技术问题诊断
  - 中等规模文档生成

### 3. Claude 3 Haiku
- **定位**：快速响应型模型
- **特点**：
  - 最快的响应速度
  - 简洁直接的答案
  - 轻量级任务处理
- **适用场景**：
  - 快速API查询
  - 简单代码片段生成
  - 语法检查
  - 快速问答

## 二、文档生成能力对比

### 1. 学习文档生成最佳选择：Claude 3 Opus
- **优势**：
  - 最强的上下文连贯性
  - 结构化思维能力强
  - 知识整合能力出色
  - 可以生成完整的学习路径
- **具体表现**：
  - 可以理解并关联多个知识点
  - 生成的文档层次分明
  - 举例详实且贴近实践
  - 适合生成系统性的学习资料

## 三、针对您的场景推荐

### 1. Java开发场景
- **主力模型**：Claude 3 Sonnet
  - 日常编码和调试
  - 单元测试编写
  - 代码优化建议
- **辅助模型**：
  - Opus：复杂功能设计
  - Haiku：快速API查询

### 2. MySQL脚本开发
- **主力模型**：Claude 3 Sonnet
  - SQL查询优化
  - 数据库设计
  - 性能调优建议
- **辅助模型**：
  - Opus：复杂查询设计
  - Haiku：简单语法确认

### 3. Shell脚本开发
- **主力模型**：Claude 3 Sonnet
  - 脚本编写和调试
  - 自动化任务实现
  - 系统管理脚本
- **辅助模型**：
  - Haiku：命令语法查询
  - Opus：复杂流程自动化

### 4. 软考备考
- **主力模型**：Claude 3 Opus
  - 知识体系构建
  - 重点难点分析
  - 真题解析和总结
  - 模拟试题生成
- **辅助模型**：
  - Sonnet：练习题讲解
  - Haiku：快速知识点回顾

## 四、使用建议

### 1. 日常开发
1. **代码编写流程**：
   - Opus：系统设计和架构规划
   - Sonnet：具体功能实现
   - Haiku：简单修改和查询

2. **问题解决流程**：
   - Haiku：快速定位问题
   - Sonnet：分析和解决问题
   - Opus：复杂问题攻关

### 2. 学习备考
1. **学习规划**：
   - Opus：制定整体学习计划
   - Sonnet：章节学习和练习
   - Haiku：零散知识点巩固

2. **复习策略**：
   - Opus：知识体系整理
   - Sonnet：重点题型练习
   - Haiku：考前快速复习

### 3. 效率优化建议
- 合理组合使用三个模型
- 根据任务复杂度选择合适模型
- 优先使用Sonnet处理日常任务
- 复杂任务交给Opus处理
- 简单查询使用Haiku

## 五、注意事项
1. 模型选择要根据具体任务复杂度
2. 重要代码生成优先使用Opus或Sonnet
3. 文档生成优先使用Opus
4. 及时总结和积累最佳实践
