~~~
if (a > 0 && b < 10) {
    语句1;
} else {
    语句2;
}
~~~

### 1.语句覆盖

- 每条语句至少执行一次，最弱、最简单。

### 2.判定覆盖

- 看整体结果 T/F

### 3.条件覆盖

- 看每个条件自己 T/F

### 4.判定/条件覆盖

- 判定 / 条件覆盖 = 同时满足两个要求：
- 每个判定的整体结果取过 T 和 F（满足判定覆盖）
- 每个条件的结果也都取过 T 和 F（满足条件覆盖）

~~~
1. 判定 / 条件覆盖（只需要 2 个用例）

a=1, b=5 → T T → 判定 T
a=-1,b=15 → F F → 判定 F
~~~

~~~
2. 条件组合覆盖（必须 4 个用例）

T T
T F
F T
F F
~~~


### 5.条件组合覆盖

- 把所有条件的「真假组合」全部覆盖一遍。
- 只要满足条件组合覆盖，前面所有覆盖都自动满足。

### 6.路径覆盖

- 程序里每一个可能的流程走向，都至少执行一次。

### 7.基本路径测试

~~~
if (a > 0) {
    语句1;
}
if (b > 0) {
    语句2;
}
语句3;
~~~

- 3条基本路径
- a>0 真 → b>0 真
- a>0 真 → b>0 假
- a>0 假 → b>0 假
- 区别
- 路径覆盖：走所有可能路径（可能爆炸多）
- 基本路径测试：走最少独立路径，覆盖所有语句和边

### 8.循环覆盖

- 让循环分别执行 0 次、1 次、多次，覆盖所有循环行为。
