- 海明校验码

### 1. 特点

- m是数据位的位数
- k是校验位的位数
- 公式为：
- $ m + k <= 2^k - 1 $

### 2. 计算校验位数

- 加入传输的尾数是9位,m=9
- 解出不等式
- $ m + k <= 2^k - 1 $
- $ k>=4 $
- 因此，总共需要传输的位数是13位
    - 9个数据位 m
    - 4个校验位 k

### 3. 校验位的位置

- 2的n次方
- $ 2^n $
    - 1,2,4,8

### 4. 计算校验位的值

- 校验位的值是通过对特定位置的数据位进行异或运算得到的。
- 具体来说，
- 每个校验位都会校验那些
    - 索引号为 校验位索引号 之和
    - 的数据位。

#### 4.1 分析为什么

- 每个信息位数拆分成n个校验位数之和
- 基础的几个十进制的值和对应的二进制表示

| 校验位坐标 | 校验位位数 | 二进制  |
|-------|-------|------|
| r0    | 1     | 0001 |
| r1    | 2     | 0010 |
| r2    | 4     | 0100 |
| r3    | 8     | 1000 |

- 举例 7
- 0111
- 所以说 校验位相加组合
    - 只有这一种可能性

| 十进制 | 二进制  |
|-----|------|
| 1   | 0001 |
| 2   | 0010 |
| 4   | 0100 |
| =   | =    |
| 7   | 0111 |

#### 4.2 信息位，拆分成n个校验位数之和

- 举例 1011
- 校验位 1 2 4
- 信息位 3 5 6 7
    - 7 = 4 + 2 + 1
    - 6 = 4 + 2
    - 5 = 4 + 1
    - 3 = 2 + 1

#### 4.3 分组

- 比如说r2是第四位，而4校验了7, 6, 5
- 所以
- r2(7, 6, 5)
- r1(7, 6, 3)
- r0(7, 5, 3)

#### 4.4 求值

- 将分组的元素转为该位置对应的信息码
- 再进行异或运算即可
- $ r2 = 1 \bigoplus 0 \bigoplus 1 = 0 $
- $ r1 = 1 \bigoplus 0 \bigoplus 1 = 0 $
- $ r0 = 1 \bigoplus 1 \bigoplus 1 = 1 $
- 输出
- r2 = 0
- r1 = 0
- r0 = 1

#### 4.5 结果表

- 信息 1011

| 7 | 6 | 5 | 4  | 3 | 2  | 1  | 位数   |
|---|---|---|----|---|----|----|------|
| 1 | 0 | 1 |    | 1 |    |    | 信息位  |
|   |   |   | r2 |   | r1 | r0 | 校验位  |
|   |   |   | 0  |   | 0  | 1  | 校验位值 |
| 1 | 0 | 1 | 0  | 1 | 0  | 1  | 最终结果 |

- 结果
- 1010101
