# 14.1. 算法的特性

## 概述
算法是解决特定问题的一系列步骤或指令的有限序列。算法是计算机科学的核心概念，是程序设计的基础。一个良好的算法应该具备五个基本特性：有穷性、确定性、可行性、输入和输出。

## 算法的定义

### 基本概念
算法（Algorithm）是解决特定问题的一系列步骤或指令的有限序列，它描述了如何从给定的输入得到期望的输出。

### 算法的特征
1. **有限性**：算法必须在有限的步骤内结束
2. **确定性**：算法的每一步都必须有明确的定义
3. **可行性**：算法的每一步都必须是可执行的
4. **输入**：算法有零个或多个输入
5. **输出**：算法有一个或多个输出

## 算法的五个基本特性

### 1. 有穷性（Finiteness）
- **定义**：算法必须在有限的步骤内结束
- **要求**：不能无限循环或无限递归
- **意义**：确保算法能够在合理时间内完成

#### 示例
```java
// 有穷性示例：计算1到n的和
public int sum(int n) {
    int result = 0;
    for (int i = 1; i <= n; i++) {  // 有限次循环
        result += i;
    }
    return result;  // 有限步骤内结束
}
```

### 2. 确定性（Definiteness）
- **定义**：算法的每一步都必须有明确的定义
- **要求**：相同的输入必须产生相同的输出
- **意义**：确保算法的行为可预测和可重复

#### 示例
```java
// 确定性示例：计算两个数的最大值
public int max(int a, int b) {
    if (a > b) {
        return a;  // 明确的条件和结果
    } else {
        return b;  // 明确的条件和结果
    }
}
```

### 3. 可行性（Effectiveness）
- **定义**：算法的每一步都必须是可执行的
- **要求**：每一步都必须在有限时间内完成
- **意义**：确保算法在理论上和实际上都是可行的

#### 示例
```java
// 可行性示例：交换两个变量的值
public void swap(int[] arr, int i, int j) {
    int temp = arr[i];  // 可执行的操作
    arr[i] = arr[j];    // 可执行的操作
    arr[j] = temp;      // 可执行的操作
}
```

### 4. 输入（Input）
- **定义**：算法有零个或多个输入
- **要求**：输入是算法处理的数据
- **意义**：算法需要数据来产生结果

#### 示例
```java
// 输入示例：查找数组中的最大值
public int findMax(int[] arr) {  // 输入：数组arr
    if (arr.length == 0) {
        throw new IllegalArgumentException("数组不能为空");
    }
    
    int max = arr[0];
    for (int i = 1; i < arr.length; i++) {
        if (arr[i] > max) {
            max = arr[i];
        }
    }
    return max;  // 输出：最大值
}
```

### 5. 输出（Output）
- **定义**：算法有一个或多个输出
- **要求**：输出是算法处理的结果
- **意义**：算法必须产生有意义的结果

#### 示例
```java
// 输出示例：排序算法
public void bubbleSort(int[] arr) {  // 输入：未排序数组
    int n = arr.length;
    for (int i = 0; i < n - 1; i++) {
        for (int j = 0; j < n - i - 1; j++) {
            if (arr[j] > arr[j + 1]) {
                swap(arr, j, j + 1);
            }
        }
    }
    // 输出：排序后的数组（通过引用传递）
}
```

## 算法的表示方法

### 1. 自然语言描述
使用人类语言描述算法的步骤。

#### 示例：冒泡排序算法
1. 比较相邻的元素，如果第一个比第二个大，就交换它们
2. 对每一对相邻元素做同样的工作，从开始第一对到结尾的最后一对
3. 重复以上步骤，直到没有任何一对数字需要比较

### 2. 伪代码
使用类似编程语言但更接近自然语言的描述方式。

#### 示例：冒泡排序伪代码
```
算法：冒泡排序
输入：数组A[1..n]
输出：排序后的数组A

1. for i = 1 to n-1 do
2.     for j = 1 to n-i do
3.         if A[j] > A[j+1] then
4.             交换A[j]和A[j+1]
5.         end if
6.     end for
7. end for
```

### 3. 流程图
使用图形符号表示算法的逻辑流程。

### 4. 程序代码
使用具体的编程语言实现算法。

## 算法的分类

### 1. 按问题类型分类
- **数值算法**：解决数值计算问题
- **非数值算法**：解决非数值计算问题

### 2. 按算法策略分类
- **分治法**：将问题分解为子问题
- **动态规划**：通过子问题的最优解求解原问题
- **贪心法**：每步都选择当前最优解
- **回溯法**：通过试错和回溯求解问题

### 3. 按时间复杂度分类
- **常数时间**：O(1)
- **对数时间**：O(log n)
- **线性时间**：O(n)
- **多项式时间**：O(n^k)
- **指数时间**：O(2^n)

## 算法设计原则

### 1. 正确性
- 算法必须能够正确解决问题
- 对于所有合法输入都能产生正确输出

### 2. 效率性
- 算法应该具有合理的时间复杂度
- 算法应该具有合理的空间复杂度

### 3. 可读性
- 算法应该易于理解和实现
- 代码应该清晰、简洁

### 4. 健壮性
- 算法应该能够处理异常情况
- 算法应该对非法输入有适当的处理

### 5. 通用性
- 算法应该适用于一类问题
- 算法应该具有较好的扩展性

## 算法分析

### 1. 时间复杂度
分析算法执行时间随输入规模增长的变化趋势。

### 2. 空间复杂度
分析算法所需内存空间随输入规模增长的变化趋势。

### 3. 最优性
分析算法是否能够找到问题的最优解。
