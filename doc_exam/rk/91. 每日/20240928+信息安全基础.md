### 1️⃣ **6.1.1. 加密技术与认证技术知识点概述**

#### 🔐 **信息安全基本概念**

##### **CIA三要素**
- **机密性（Confidentiality）**：信息不被未授权访问
- **完整性（Integrity）**：信息不被未授权修改
- **可用性（Availability）**：授权用户能正常访问信息

##### **信息安全威胁**
- **被动攻击**：窃听、流量分析
- **主动攻击**：篡改、伪造、重放、拒绝服务

#### 🛡️ **安全服务分类**

- **认证服务**：身份认证、消息认证
- **访问控制**：强制访问控制、自主访问控制
- **数据机密性**：连接机密性、无连接机密性
- **数据完整性**：连接完整性、无连接完整性
- **不可否认性**：发送方不可否认、接收方不可否认

#### 🔑 **密码学基础**

##### **密码系统组成**
- **明文（Plaintext）**：原始信息
- **密文（Ciphertext）**：加密后的信息
- **密钥（Key）**：加密/解密的参数
- **加密算法（Encryption）**：明文→密文的过程
- **解密算法（Decryption）**：密文→明文的过程

##### **密码分析**
- **穷举攻击**：尝试所有可能密钥
- **统计分析**：基于密文的统计特性
- **差分分析**：分析输入差异对输出的影响
- **线性分析**：寻找加密算法的线性逼近

### 2️⃣ **6.1.2. 对称与非对称加密技术**

#### 🔄 **对称加密技术**

##### **基本原理**
- **特点**：加密密钥 = 解密密钥
- **速度**：加密解密速度快
- **密钥管理**：密钥分发困难
- **适用场景**：大量数据加密

##### **DES（数据加密标准）**
- **密钥长度**：56位（实际64位，8位奇偶校验）
- **分组长度**：64位
- **轮数**：16轮Feistel结构
- **安全性**：已被破解，不推荐使用
- **变种**：3DES（三重DES，112位或168位密钥）

```
DES加密过程：
明文(64位) → 初始置换 → 16轮Feistel → 逆初始置换 → 密文(64位)
```

##### **AES（高级加密标准）**
- **密钥长度**：128位、192位、256位
- **分组长度**：128位
- **轮数**：10轮（AES-128）、12轮（AES-192）、14轮（AES-256）
- **结构**：SPN（置换-置换网络）
- **安全性**：目前最安全的对称加密算法

```
AES算法结构：
1. 字节替代（SubBytes）
2. 行位移（ShiftRows）  
3. 列混合（MixColumns）
4. 轮密钥加（AddRoundKey）
```

##### **其他对称算法**
- **RC4**：流密码，WEP/WPA中使用
- **Blowfish**：32-448位可变密钥长度
- **Twofish**：AES候选算法之一

#### 🔐 **非对称加密技术**

##### **基本原理**
- **特点**：加密密钥 ≠ 解密密钥
- **密钥对**：公钥（公开）+ 私钥（保密）
- **速度**：比对称加密慢100-1000倍
- **适用场景**：密钥交换、数字签名、小量数据加密

##### **RSA算法**
- **原理**：基于大整数因式分解困难性
- **密钥长度**：1024位、2048位、4096位
- **用途**：加密、数字签名、密钥交换
- **安全性**：目前最广泛使用的非对称算法

```
RSA加密过程：
1. 选择两个大素数p、q
2. 计算n = p × q，φ(n) = (p-1)(q-1)
3. 选择公钥指数e，满足gcd(e,φ(n))=1
4. 计算私钥指数d，满足ed ≡ 1 (mod φ(n))
5. 公钥：(n,e)，私钥：(n,d)
6. 加密：C = M^e mod n
7. 解密：M = C^d mod n
```

##### **DSA（数字签名算法）**
- **原理**：基于离散对数问题
- **用途**：仅用于数字签名，不能加密
- **密钥长度**：1024位、2048位、3072位
- **标准**：NIST FIPS 186-4

##### **ECC（椭圆曲线密码）**
- **原理**：基于椭圆曲线离散对数问题
- **优势**：相同安全强度下密钥长度更短
- **效率**：计算效率高，适合移动设备
- **对比**：256位ECC ≈ 3072位RSA

#### 📊 **对称vs非对称加密对比**

| 特性 | 对称加密 | 非对称加密 |
|------|---------|-----------|
| 密钥关系 | 加密密钥=解密密钥 | 加密密钥≠解密密钥 |
| 速度 | 快（MB/s级别） | 慢（KB/s级别） |
| 密钥管理 | 困难（n²个密钥） | 简单（2n个密钥） |
| 密钥长度 | 较短（128-256位） | 较长（1024-4096位） |
| 典型算法 | DES、AES、RC4 | RSA、DSA、ECC |
| 主要用途 | 大量数据加密 | 密钥交换、数字签名 |
| 安全基础 | 密钥保密 | 数学难题 |

#### 🔗 **混合加密系统**

```
实际应用中的组合：
1. 用非对称加密传输对称密钥
2. 用对称加密传输大量数据
3. 兼顾安全性和效率
```

### 3️⃣ **6.1.3. 数字签名与信息摘要**

#### ✍️ **数字签名技术**

##### **基本概念**
- **定义**：用私钥对消息进行签名，用公钥验证签名
- **目的**：消息认证、身份认证、不可否认性
- **原理**：只有私钥持有者能生成有效签名

##### **数字签名过程**

```
签名过程：
明文 → 哈希函数 → 消息摘要 → 私钥加密 → 数字签名

验证过程：
数字签名 → 公钥解密 → 消息摘要1
明文 → 哈希函数 → 消息摘要2
比较摘要1和摘要2是否相同
```

##### **数字签名特性**
- **真实性**：确认签名者身份
- **完整性**：确保数据未被篡改
- **不可否认性**：签名者不能否认
- **不可伪造性**：他人无法伪造签名

#### 🏷️ **消息摘要（哈希函数）**

##### **基本特性**
- **单向性**：由摘要无法反推原文
- **确定性**：相同输入产生相同输出
- **雪崩效应**：输入微小变化导致输出巨大变化
- **抗碰撞性**：很难找到产生相同摘要的不同输入

##### **MD5算法**
- **摘要长度**：128位（32个十六进制字符）
- **分组长度**：512位
- **安全性**：已被破解，存在碰撞攻击
- **应用**：文件完整性校验（不推荐安全应用）

```
MD5特点：
输入：任意长度消息
输出：128位摘要
速度：快速计算
问题：存在碰撞攻击
```

##### **SHA系列算法**

| 算法 | 摘要长度 | 分组长度 | 安全性 | 应用场景 |
|------|---------|---------|--------|---------|
| SHA-1 | 160位 | 512位 | 已破解 | 不推荐 |
| SHA-256 | 256位 | 512位 | 安全 | 推荐使用 |
| SHA-384 | 384位 | 1024位 | 安全 | 高安全场景 |
| SHA-512 | 512位 | 1024位 | 安全 | 高安全场景 |

##### **哈希函数应用**
- **数字签名**：对摘要签名而非原文
- **密码存储**：存储密码的哈希值
- **文件校验**：验证文件完整性
- **区块链**：比特币等区块链技术
- **HMAC**：基于哈希的消息认证码

#### 🔐 **消息认证码（MAC）**

##### **HMAC原理**
```
HMAC(K,M) = H((K ⊕ opad) || H((K ⊕ ipad) || M))
其中：
K：密钥
M：消息
H：哈希函数
opad：外部填充
ipad：内部填充
```

##### **MAC vs 数字签名**
- **MAC**：共享密钥，双方认证，无不可否认性
- **数字签名**：公私钥对，单方认证，有不可否认性

### 4️⃣ **6.1.4. 数字证书应用**

#### 📜 **数字证书概念**

##### **定义和作用**
- **定义**：由权威机构签发的电子身份证明
- **作用**：绑定实体身份与公钥
- **目的**：解决公钥分发和认证问题

##### **证书内容（X.509标准）**
```
数字证书包含：
1. 版本号
2. 序列号
3. 签名算法标识
4. 颁发者（CA）信息
5. 有效期（起始时间-结束时间）
6. 主体（证书持有者）信息
7. 主体公钥信息
8. 扩展字段
9. CA的数字签名
```

#### 🏛️ **PKI公钥基础设施**

##### **PKI组成**
- **CA（认证中心）**：颁发和管理证书
- **RA（注册中心）**：身份审核和证书申请处理
- **证书库**：存储和分发证书
- **密钥备份与恢复**：密钥管理服务
- **证书撤销列表（CRL）**：已撤销证书清单

##### **CA层次结构**
```
根CA（Root CA）
├── 中间CA1（Intermediate CA）
│   ├── 子CA1-1
│   └── 子CA1-2
└── 中间CA2
    ├── 子CA2-1
    └── 子CA2-2
```

#### 🔗 **证书链验证**

##### **证书链概念**
- **定义**：从用户证书到根CA证书的信任路径
- **验证过程**：逐级验证每个证书的有效性
- **信任锚点**：预置的根CA证书

##### **验证步骤**
```
1. 检查证书格式和语法
2. 检查证书有效期
3. 检查证书撤销状态（CRL/OCSP）
4. 验证证书签名
5. 检查证书使用限制
6. 构建并验证证书链
7. 检查信任锚点
```

#### 📱 **证书类型和应用**

##### **按验证级别分类**
- **DV证书**：域名验证，验证域名所有权
- **OV证书**：组织验证，验证组织身份
- **EV证书**：扩展验证，最高级别验证

##### **按应用场景分类**
- **SSL/TLS证书**：网站HTTPS加密
- **代码签名证书**：软件完整性验证
- **电子邮件证书**：邮件加密和签名
- **客户端证书**：用户身份认证

### 5️⃣ **6.2. 网络安全协议**

#### 🔒 **SSL/TLS协议**

##### **协议概述**
- **SSL**：安全套接字层（已废弃）
- **TLS**：传输层安全协议（SSL继承者）
- **版本**：TLS 1.0/1.1（已废弃）、TLS 1.2、TLS 1.3
- **端口**：HTTPS（443）、SMTPS（465）、POP3S（995）

##### **TLS协议栈**
```
应用层（HTTP、SMTP等）
├── TLS记录协议
├── TLS握手协议
├── TLS更改密码规范协议
└── TLS警报协议
TCP传输层
```

##### **TLS握手过程**
```
客户端                     服务器
1. Client Hello          →
                          ← 2. Server Hello
                          ← 3. Certificate
                          ← 4. Server Key Exchange
                          ← 5. Server Hello Done
6. Client Key Exchange    →
7. Change Cipher Spec     →
8. Finished              →
                          ← 9. Change Cipher Spec
                          ← 10. Finished
```

##### **TLS安全特性**
- **机密性**：对称加密保护数据
- **完整性**：MAC验证数据完整性
- **认证性**：证书验证服务器身份
- **前向安全**：会话密钥独立性

#### 🌐 **HTTPS协议**

##### **HTTP vs HTTPS**
- **HTTP**：明文传输，端口80，无安全保障
- **HTTPS**：SSL/TLS加密，端口443，安全传输

##### **HTTPS工作流程**
```
1. 客户端发起HTTPS请求
2. 服务器返回证书
3. 客户端验证证书
4. TLS握手建立安全通道
5. 加密HTTP通信
6. 服务器返回加密响应
7. 客户端解密显示内容
```

#### 🔐 **IPSec协议**

##### **IPSec概述**
- **定义**：IP层安全协议套件
- **位置**：网络层（IP层）
- **模式**：传输模式、隧道模式
- **服务**：认证、完整性、机密性

##### **IPSec组成**
- **AH（认证头）**：提供认证和完整性
- **ESP（封装安全载荷）**：提供认证、完整性、机密性
- **IKE（密钥交换）**：密钥管理协议

##### **IPSec模式**
```
传输模式：
原IP头 | IPSec头 | 数据载荷 | IPSec尾

隧道模式：
新IP头 | IPSec头 | 原IP头 | 数据载荷 | IPSec尾
```

#### 🔑 **SSH协议**

##### **SSH概述**
- **定义**：安全外壳协议
- **端口**：22
- **用途**：远程登录、文件传输、端口转发
- **版本**：SSH-1（已废弃）、SSH-2

##### **SSH认证方式**
- **密码认证**：用户名+密码
- **公钥认证**：公私钥对认证
- **主机认证**：基于主机的认证

##### **SSH相关协议**
- **SCP**：基于SSH的文件复制
- **SFTP**：SSH文件传输协议
- **SSH隧道**：端口转发和代理

#### 📧 **邮件安全协议**

##### **S/MIME**
- **定义**：安全多用途互联网邮件扩展
- **功能**：邮件加密、数字签名
- **标准**：PKCS#7、X.509证书

##### **PGP（Pretty Good Privacy）**
- **特点**：端到端加密
- **密钥管理**：信任网络模型
- **应用**：个人邮件安全
