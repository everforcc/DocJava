### 1️⃣ **5.4.1. IP地址与子网划分概述**

#### 🌐 **IP地址基础概念**

- **定义**：Internet Protocol Address，网络层地址标识符
- **作用**：唯一标识网络中的每台设备
- **格式**：32位二进制数，通常用点分十进制表示
- **组成**：网络部分 + 主机部分
- **版本**：IPv4（32位）和IPv6（128位）

#### 📊 **IP地址的表示方法**

```
二进制表示：11000000.10101000.00000001.00000001
十进制表示：192.168.1.1
十六进制表示：C0.A8.01.01
```

#### 🎯 **子网划分的意义**

- **提高地址利用率**：避免地址浪费
- **减少广播域**：提升网络性能
- **便于网络管理**：逻辑分组管理
- **增强网络安全**：隔离不同网段

### 2️⃣ **5.4.2. IP地址分类与特殊的IP地址**

#### 📋 **IP地址分类详解**

| 类型 | 范围                          | 网络位 | 主机位 | 默认子网掩码        | 网络数量       | 主机数量        |
|----|-----------------------------|-----|-----|---------------|------------|-------------|
| A类 | 1.0.0.0 - 126.255.255.255   | 8位  | 24位 | 255.0.0.0     | 126个       | 16,777,214个 |
| B类 | 128.0.0.0 - 191.255.255.255 | 16位 | 16位 | 255.255.0.0   | 16,384个    | 65,534个     |
| C类 | 192.0.0.0 - 223.255.255.255 | 24位 | 8位  | 255.255.255.0 | 2,097,152个 | 254个        |
| D类 | 224.0.0.0 - 239.255.255.255 | -   | -   | -             | 组播地址       | -           |
| E类 | 240.0.0.0 - 255.255.255.255 | -   | -   | -             | 实验用        | -           |

#### 🔒 **私有IP地址段（RFC 1918）**

- **A类私有**：10.0.0.0/8（10.0.0.0 - 10.255.255.255）
- **B类私有**：172.16.0.0/12（172.16.0.0 - 172.31.255.255）
- **C类私有**：192.168.0.0/16（192.168.0.0 - 192.168.255.255）

#### ⭐ **特殊IP地址**

- **127.0.0.1**：本地回环地址（localhost）
- **0.0.0.0**：默认路由地址
- **255.255.255.255**：有限广播地址
- **169.254.x.x**：APIPA自动私有IP地址
- **224.0.0.1**：所有主机组播地址
- **224.0.0.2**：所有路由器组播地址

#### 🚫 **不可用的IP地址**

- **网络地址**：主机位全为0
- **广播地址**：主机位全为1
- **回环地址**：127.x.x.x
- **组播地址**：224.x.x.x - 239.x.x.x

### 3️⃣ **5.4.3. 子网划分与路由汇聚**

#### 📐 **子网掩码原理**

- **作用**：区分网络部分和主机部分
- **格式**：连续的1表示网络位，连续的0表示主机位
- **CIDR表示法**：192.168.1.0/24（24个网络位）

#### 🔢 **子网划分计算方法**

##### **步骤1：确定需求**

- 需要多少个子网？
- 每个子网需要多少台主机？

##### **步骤2：计算借位数**

- 子网数 ≤ 2^借位数
- 主机数 ≤ 2^(主机位数) - 2

##### **步骤3：确定新的子网掩码**

- 原网络位 + 借位数 = 新网络位

##### **步骤4：列出子网范围**

#### 💡 **子网划分实战示例**

**例题1：192.168.1.0/24 划分为8个子网**

```
原网络：192.168.1.0/24
借位数：3位（2^3 = 8个子网）
新掩码：/27（24+3）即255.255.255.224
子网增量：32（256-224）

子网列表：
1. 192.168.1.0/27    (0-31)
2. 192.168.1.32/27   (32-63)
3. 192.168.1.64/27   (64-95)
4. 192.168.1.96/27   (96-127)
5. 192.168.1.128/27  (128-159)
6. 192.168.1.160/27  (160-191)
7. 192.168.1.192/27  (192-223)
8. 192.168.1.224/27  (224-255)
```

#### 🔄 **路由汇聚（超网）**

- **目的**：减少路由表项，提高路由效率
- **原理**：将多个连续的网络合并为一个更大的网络
- **要求**：网络地址必须连续且符合2的幂次方

**汇聚示例**：

```
192.168.0.0/24
192.168.1.0/24
192.168.2.0/24
192.168.3.0/24
汇聚为：192.168.0.0/22
```

### 4️⃣ **5.4.4. IPv6基础**

#### 🌟 **IPv6概述**

- **地址长度**：128位（IPv4为32位）
- **地址空间**：约3.4×10^38个地址
- **表示方法**：8组4位十六进制数，用冒号分隔
- **示例**：2001:0db8:85a3:0000:0000:8a2e:0370:7334

#### 📝 **IPv6地址表示**

##### **完整表示**

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

##### **省略前导零**

```
2001:db8:85a3:0:0:8a2e:370:7334
```

##### **压缩连续零**

```
2001:db8:85a3::8a2e:370:7334
```

#### 🏷️ **IPv6地址类型**

- **单播地址**：一对一通信
- **组播地址**：一对多通信（FF00::/8）
- **任播地址**：一对最近一个通信
- **本地链路地址**：FE80::/10

#### 🆚 **IPv6与IPv4对比**

| 特性   | IPv4      | IPv6      |
|------|-----------|-----------|
| 地址长度 | 32位       | 128位      |
| 地址表示 | 点分十进制     | 冒号分隔十六进制  |
| 地址空间 | 43亿       | 3.4×10^38 |
| 头部长度 | 可变20-60字节 | 固定40字节    |
| 自动配置 | DHCP      | 无状态自动配置   |
| 安全性  | 可选        | 内置IPSec   |

### 5️⃣ **5.5. 网络规划与设计**

#### 🎯 **网络规划原则**

- **可扩展性**：支持未来业务增长
- **可靠性**：冗余设计，避免单点故障
- **安全性**：访问控制，数据加密
- **性能优化**：带宽规划，负载均衡
- **易管理性**：统一管理，监控告警

#### 📊 **网络拓扑设计**

##### **星型拓扑**

- **优点**：故障隔离好，易于管理
- **缺点**：中心节点依赖性强
- **适用**：小型办公网络

##### **树型拓扑**

- **优点**：层次清晰，易于扩展
- **缺点**：根节点压力大
- **适用**：企业网络

##### **环型拓扑**

- **优点**：冗余路径，可靠性高
- **缺点**：配置复杂
- **适用**：核心网络

#### 🏢 **网络分层设计**

- **核心层**：高速转发，路由聚合
- **汇聚层**：策略控制，负载分担
- **接入层**：用户接入，安全控制

### 6️⃣ **5.6. 网络接入技术**

#### 📡 **有线接入技术**

##### **ADSL（非对称数字用户线）**

- **下行速率**：1-8Mbps
- **上行速率**：128Kbps-1Mbps
- **传输介质**：电话线
- **优点**：利用现有线路，成本低
- **缺点**：距离限制，速率不对称

##### **光纤接入**

- **FTTH**：光纤到户
- **FTTB**：光纤到楼
- **FTTC**：光纤到路边
- **优点**：带宽大，传输距离远
- **缺点**：建设成本高

##### **以太网接入**

- **LAN**：局域网接入
- **速率**：10Mbps/100Mbps/1Gbps/10Gbps
- **介质**：双绞线、光纤

#### 📶 **无线接入技术**

##### **WiFi（802.11）**

- **802.11a**：5GHz，54Mbps
- **802.11b**：2.4GHz，11Mbps
- **802.11g**：2.4GHz，54Mbps
- **802.11n**：2.4/5GHz，300Mbps
- **802.11ac**：5GHz，1Gbps+

##### **蜂窝移动通信**

- **2G**：GSM，9.6Kbps
- **3G**：WCDMA/CDMA2000，2Mbps
- **4G**：LTE，100Mbps
- **5G**：NR，1Gbps+

### 7️⃣ **5.7.1-5.7.3. WWW服务全套**

#### 🌍 **HTTP协议基础**

##### **HTTP特点**

- **无状态**：每次请求独立
- **基于TCP**：可靠传输
- **端口80**：默认HTTP端口
- **端口443**：HTTPS端口

##### **HTTP请求方法**

- **GET**：获取资源
- **POST**：提交数据
- **PUT**：更新资源
- **DELETE**：删除资源
- **HEAD**：获取头信息
- **OPTIONS**：查询支持的方法

##### **HTTP状态码**

- **1xx**：信息性状态码
- **2xx**：成功状态码（200 OK）
- **3xx**：重定向状态码（301, 302）
- **4xx**：客户端错误（404 Not Found）
- **5xx**：服务器错误（500 Internal Server Error）

#### 🔗 **URL结构分析**

```
https://www.example.com:443/path/to/resource?param=value#section

协议:// 主机名 :端口 /路径 ?查询参数 #锚点
```

- **协议**：http, https, ftp等
- **主机名**：域名或IP地址
- **端口**：服务端口号
- **路径**：资源路径
- **查询参数**：传递给服务器的参数
- **锚点**：页面内定位

#### 🖥️ **Web服务器配置要点**

##### **Apache配置**

- **DocumentRoot**：网站根目录
- **ServerName**：服务器名称
- **VirtualHost**：虚拟主机
- **Directory**：目录权限

##### **Nginx配置**

- **server**：服务器块
- **location**：位置匹配
- **upstream**：负载均衡
- **proxy_pass**：反向代理

##### **IIS配置**

- **站点管理**：创建网站
- **应用程序池**：进程隔离
- **绑定**：域名和端口
- **身份验证**：访问控制
