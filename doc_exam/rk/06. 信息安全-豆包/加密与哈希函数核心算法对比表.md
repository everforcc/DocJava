### 对称加密

| 类型           | 算法名称 | 核心参数                                                   | 安全性                 | 典型应用场景                            | 软考高频考点                                       |
| -------------- | -------- | ---------------------------------------------------------- | ---------------------- | --------------------------------------- | -------------------------------------------------- |
| **对称加密**   | DES      | 密钥长度 56bit，分组长度 64bit                             | 已破解，不安全         | 早期数据加密（已淘汰）                  | 考查其淘汰原因，作为对称加密的历史算法考点         |
|                | 3DES     | 密钥长度 112/168bit，分组长度 64bit                        | 安全，但效率低         | 兼容 DES 的 legacy 系统                 | 考查其与 DES 的关系，安全性提升方式                |
|                | AES      | 密钥长度 128/192/256bit，分组长度 128bit                   | 安全，当前主流         | 数据传输加密、文件存储、VPN             | 考查其作为对称加密标准的地位，密钥长度选项         |
|                | SM4      | 密钥长度 128bit，分组长度 128bit                           | 安全，等效 AES-128     | 国产信创项目、政务系统、WAPI            | 软考新增考点，考查国产算法属性及应用               |

### 非对称加密

| 类型           | 算法名称 | 核心参数                                                   | 安全性                 | 典型应用场景                            | 软考高频考点                                       |
| -------------- | -------- | ---------------------------------------------------------- | ---------------------- | --------------------------------------- | -------------------------------------------------- |
| **非对称加密** | RSA      | 密钥长度 512/1024/2048/4096bit，基于大素数分解             | 安全，2048bit 以上推荐 | 数字签名、密钥交换、证书加密            | 考查其算法原理（大素数分解），应用场景             |
|                | ECC      | 密钥长度 160/256bit（等效 RSA 1024/3072bit），基于椭圆曲线 | 安全，密钥短、效率高   | 移动设备、物联网、区块链                | 考查其与 RSA 的效率对比，高安全性 + 低资源消耗特点 |
|                | SM2      | 密钥长度 256bit，基于椭圆曲线                              | 安全，国产合规         | 国产数字签名、身份认证、密钥交换        | 软考新增考点，考查与 SM3/SM4 的协同应用            |
|                | DSA      | 密钥长度 1024/2048bit，基于离散对数                        | 安全，仅用于签名       | 早期数字签名场景（逐步被 RSA/ECC 替代） | 考查其 “仅签名” 的特性，与 RSA 的功能区别          |

### 哈希函数

| 类型           | 算法名称 | 核心参数                                                   | 安全性                 | 典型应用场景                            | 软考高频考点                                       |
| -------------- | -------- | ---------------------------------------------------------- | ---------------------- | --------------------------------------- | -------------------------------------------------- |
| **哈希函数**   | MD5      | 输出长度 128bit                                            | 已破解，存在碰撞漏洞   | 非安全场景的文件校验（如本地文件比对）  | 考查其安全缺陷，作为反面案例考点                   |
|                | SHA-1    | 输出长度 160bit                                            | 已破解，碰撞攻击成功   | 早期版本控制、数字签名（已淘汰）        | 考查其被 SHA-2 替代的原因                          |
|                | SHA-256  | 输出长度 256bit                                            | 安全，当前主流         | 数字签名、区块链（比特币）、HTTPS       | 考查其应用场景，输出长度记忆                       |
|                | SHA-512  | 输出长度 512bit                                            | 安全，高安全等级       | 金融、军事等高敏感领域                  | 考查与 SHA-256 的安全性、效率对比                  |
|                | SHA-3    | 输出长度可变（224/256/384/512bit）                         | 安全，抗量子攻击       | 未来安全场景、量子计算适配              | 考查其海绵结构，抗量子攻击特性                     |
|                | SM3      | 输出长度 256bit                                            | 安全，等效 SHA-256     | 国产政务系统、数字签名（搭配 SM2）      | 软考新增考点，国产算法属性及协同应用               |
|                | CRC32    | 输出长度 32bit                                             | 无抗碰撞性，仅用于校验 | 数据传输错误检测（如网络数据包）        | 考查其与安全哈希函数的区别（干扰项）               |
