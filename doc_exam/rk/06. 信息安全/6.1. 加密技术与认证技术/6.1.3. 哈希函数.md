### 哈希

- 哈希函数又称散列函数，是一种将任意长度的输入数据（明文）映射为固定长度的输出数据（哈希值 / 散列值 / 摘要）的单向数学函数，记为：h = H(M)，其中M为明文，h为哈希值。

### 四大特性

| 特性             | 定义                                                         | 考试要点                                                     |
| ---------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **单向性**       | 从明文`M`计算哈希值`h`容易，但从`h`反向推导`M`在计算上不可行 | 常考应用：密码存储（如用户密码加密存储）                     |
| **抗碰撞性**     | 分为弱抗碰撞和强抗碰撞，核心是找到两个不同明文产生相同哈希值在计算上不可行 | 弱抗碰撞：给定`M1`，难以找到`M2≠M1`使`H(M1)=H(M2)`强抗碰撞：难以找到任意`M1≠M2`使`H(M1)=H(M2)` |
| **雪崩效应**     | 明文的微小变化（如 1bit），会导致哈希值发生巨大变化          | 考查理解：为何该特性能保障数据完整性                         |
| **输出固定长度** | 无论输入数据长度如何，输出的哈希值长度固定                   | 举例：MD5 输出 128bit，SHA-1 输出 160bit，SHA-256 输出 256bit |

#### 抗碰撞性

| 对比维度     | 弱抗碰撞性                                   | 强抗碰撞性                                                   | 软考考查重点                       |
| ------------ | -------------------------------------------- | ------------------------------------------------------------ | ---------------------------------- |
| **攻击前提** | 已知一个明文和对应的哈希值                   | 无任何前提，凭空寻找                                         | 区分两种抗碰撞性的前提条件         |
| **攻击目标** | 找到另一个明文，哈希值与已知值相同           | 找到任意两个不同明文，哈希值相同                             | 判断题目描述属于哪种抗碰撞性       |
| **安全强度** | 要求较低                                     | 要求更高                                                     | 强抗碰撞性是哈希函数的核心安全指标 |
| **典型应用** | 密码存储（防止通过哈希值反向推导或伪造密码） | 数字签名、数据完整性校验（防止恶意构造两个不同内容但哈希值相同的文件） | 结合应用场景判断所需的抗碰撞性等级 |
