# 软考软件设计师学习计划（基于个人技术栈）

## 一、学习目标
- 通过软考中级《软件设计师》考试
- 掌握软件工程、系统设计、数据库、数据结构与算法、网络、操作系统等核心知识
- 提升理论基础，结合实际开发经验，强化综合能力

---

## 二、技术栈与考试内容对照
| 技术栈         | 软考知识点关联           |
| -------------- | ---------------------- |
| Java、SpringBoot | 程序设计、面向对象、UML、设计模式、开发工具、编码规范 |
| Python         | 程序设计、算法实现、脚本自动化 |
| MySQL          | 数据库原理、SQL、事务、范式、数据建模 |
| Redis          | 缓存、NoSQL、分布式基础 |
| Linux、Docker  | 操作系统、虚拟化、部署、运维 |
| Nginx          | 网络、负载均衡、服务治理 |
| HTML、Vue      | 前端基础、Web开发、B/S架构 |

---

## 三、阶段学习规划

### 第一阶段：基础理论夯实（2周）
- 计算机组成原理、操作系统、数据结构与算法、计算机网络
- 每日建议：2小时
- 重点：
  - 数据结构（线性表、树、图、查找、排序）
  - 操作系统（进程、线程、内存管理、文件系统）
  - 网络（OSI七层、TCP/IP、常用协议）
  - 计算机组成（存储、指令系统、输入输出）
- 推荐资源：
  - 王道考研系列教材
  - B站软考/计算机基础课程

### 第二阶段：软件工程与系统设计（2周）
- 软件工程、UML、面向对象、设计模式、项目管理
- 每日建议：2小时
- 重点：
  - 软件开发生命周期、需求分析、概要设计、详细设计
  - UML图（用例、类、时序、活动、状态等）
  - 常用设计模式（单例、工厂、观察者、策略等）
  - 项目管理基础（进度、质量、风险、配置管理）
- 推荐资源：
  - 《软件工程（第9版）》（Ian Sommerville）
  - 极客时间《设计模式之美》
  - 菜鸟教程UML

### 第三阶段：数据库与开发实践（1周）
- 数据库原理、SQL、事务、范式、NoSQL、开发工具
- 每日建议：1.5小时
- 重点：
  - 数据库三范式、ER建模、SQL查询优化
  - 事务、并发控制、索引、视图、存储过程
  - NoSQL基础（Redis原理、应用场景）
  - 开发工具与环境（IDE、版本管理、自动化部署）
- 推荐资源：
  - 《数据库系统概论》
  - LeetCode SQL题库
  - 菜鸟教程MySQL/Redis

### 第四阶段：真题训练与查漏补缺（1周）
- 历年真题、模拟题、错题整理、知识点回顾
- 每日建议：2小时
- 重点：
  - 上午选择题：基础理论、软件工程、数据库、网络、操作系统
  - 下午案例分析与程序设计题：算法、UML、设计模式、SQL、代码阅读
  - 查漏补缺，整理错题本
- 推荐资源：
  - 软考官网真题库
  - B站软考真题讲解
  - 微信公众号"软考一点通"

---

## 四、每日学习任务建议
- 早上/晚上固定时间学习，保持连续性
- 每天复盘当天内容，整理错题和难点
- 每周末做一次阶段性小测验
- 重点难点及时查阅资料，做好知识卡片

---

## 五、实用备考建议
- 结合实际开发经验理解理论知识，注重知识迁移
- 多做真题，熟悉考试题型和答题节奏
- 关注软考政策、报名时间、考试安排
- 考前一周以真题训练和错题复习为主，保持良好作息
