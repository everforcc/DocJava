- Product Key (PK)
- Device Name (DN)
- Device Secret (DS)
- Product Secret (PS)

### 说明

| 参数名称  | 英文缩写                | 作用（What）                                         | 级别（Level） | 描述（How）                             |
|-------|---------------------|--------------------------------------------------|-----------|-------------------------------------|
| 产品密钥  | PK (Product Key)    | 产品标识。全局唯一，用于标识一类相同型号/功能的设备。                      | 产品级       | 在云平台上创建产品时生成，用于定义一批设备的物模型、Topic权限等。 |
| 设备名称  | DN (Device Name)    | 设备标识。在所属产品内唯一，用于标识产品下的一台具体的物理设备。                 | 设备级       | 通常使用设备的唯一序列号 (SN) 或其他唯一标识符作为 DN。    |
| 产品密钥对 | PK + DN             | 设备唯一身份。两者组合起来，在全球范围内唯一标识一个设备。                    | 身份标识      | 连接云服务时，必须提供这对信息以声明“我是谁”。            |
| 设备密钥  | DS (Device Secret)  | 设备凭证/密码。用于与云平台进行身份认证和安全通信的加密密钥。                  | 设备级       | 在云平台上创建设备时生成，用于在设备端计算签名，证明设备是合法的。   |
| 产品密钥  | PS (Product Secret) | 产品凭证/密码。在**“一型一密”模式下使用，用于设备首次连接云端进行动态注册/激活**时使用。 | 产品级       | 仅在设备出厂时无法预置 DS，需要动态获取 DS 的场景中使用。    |
