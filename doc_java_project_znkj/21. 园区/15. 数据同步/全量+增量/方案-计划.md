### 数据结构

- 每个表都有
- id，内容为uuid
- create_time，内容为当前时间
- update_time，内容为更新时间
- 云端的表比本地多一个字段，sn，其他字段都一样，如果要查询出增量数据的话，怎样才能拼装为本地的新增或更新脚本

### 技术点

- docker + java + springboot + mysql

### mysql数据备份方案

- 本地设备很多，每个设备都有相同的数据库,本地设备没有ip
- 云端有同样的数据库，数据库有ip
- 本地设备每天0点全量备份文件到云端保存，0点以后的数据增量备份到云端的数据库

### 问题如何恢复数据

- 全量的可以直接恢复到数据库
- 增量的数据，有更新的，有新增的，如何恢复数据
