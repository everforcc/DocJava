### 常用实现方式

- 在Linux下，常用的I/O多路复用方式有三种：select、poll和epoll。

### select和poll

- select和poll的原理是基于轮询，
- 即不断地查询所有注册的I/O事件，如果有事件发生就立即通知应用程序。
- 这种方式的效率较低，因为每次查询都需要遍历所有的I/O事件。

### epoll

- epoll的原理是基于事件通知，
- 即只有当I/O事件发生时，才会通知应用程序。
- 这种方式的效率更高，因为它避免了无效的查询。
