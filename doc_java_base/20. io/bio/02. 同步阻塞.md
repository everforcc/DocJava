### 1.什么是同步阻塞

- 同步、阻塞是两个概念，

### 同步

- 同步，即原地傻等，在发出一个调用时，在没有得到结果之前，该调用就不返回。
- 但是一旦调用返回，就得到返回值了。
- 换句话说，就是由调用者主动等待这个调用的结果。
- 同步是对客户端的描述，在等待服务端响应的请求时，客户端不做其他的事情。

### 阻塞

- 阻塞，即卡住，是对服务端的描述。
- 服务端会不会因为本来该开始处理请求，进行后续的业务逻辑了，
- 但是因为请求的数据包还没有接收完（因为一个请求不一定是一个数据包发过来的，可能分成了好几个数据包），
- 而需要卡在原地等待网络IO将数据包接收完才能继续往下走。
