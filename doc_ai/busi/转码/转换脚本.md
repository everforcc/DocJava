# Windows 10 下批量将GBK编码文件转换为UTF-8操作流程（适用于IntelliJ IDEA项目）

## 1. 备份项目
在进行批量转换前，建议先备份你的项目文件，以防止意外情况导致数据丢失。

## 2. 打开PowerShell
在项目根目录下，按住Shift键并右键点击空白处，选择“在此处打开PowerShell窗口”。

## 3. 执行批量转换脚本
以下脚本会将当前目录及所有子目录下的`.java`、`.txt`、`.md`文件从GBK编码转换为UTF-8编码：

```powershell
Get-ChildItem -Recurse -Include *.java,*.txt,*.md | ForEach-Object {
    $content = Get-Content $_.FullName -Encoding Default
    Set-Content $_.FullName -Value $content -Encoding UTF8
}
```

- `-Include` 后可根据需要添加或删除文件类型。
- `-Encoding Default` 在中文Windows下通常对应GBK编码。
- 该脚本会**覆盖原文件**，请确保已做好备份。

## 4. IDEA中统一编码设置为UTF-8
1. 打开 IntelliJ IDEA。
2. 依次进入 `File` → `Settings` → `Editor` → `File Encodings`。
3. 将“Global Encoding”、“Project Encoding”、“Default encoding for properties files”都设置为 `UTF-8`。
4. 勾选“Transparent native-to-ascii conversion”。
5. 重启IDEA。

## 5. 验证
- 随机打开原本为GBK的文件，确认中文未出现乱码。
- 如有乱码，说明该文件原本不是GBK编码，需手动修复。

---

如需处理其他文件类型或有特殊需求，可修改脚本中的文件类型。
