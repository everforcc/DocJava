### 3. 高可用保障机制

#### 3.1 系统

- 对外暴露2台nginx服务地址，每台服务通过权重映射2个网关服务，网关通过openfeign调用服务接口。
- 每个微服务保证在注册中心至少注册2个实例，并且实例不属于同一台物理服务器，注册中心也会部署在不同的服务器

##### 项目结构示意图

- 用户-ng-不同业务服务-注册中心

#### 3.2 数据库

- 数据库采用 主-从-备
- 主节点负责读写，从节点负责读取，备节点负责定时备份数据
- 保证高可用

#### 3.3 缓存

- 使用redis作为缓存中间件
- Redis Cluster至少配置三个主节点，三个从节点
- 由于CRC16算法，主节点之间不会保存冗余数据，由从节点冗余
- 通过负载均衡，使缓存均匀的分布在不同的节点，如果主节点宕机，从节点会自动升级为主节点，保证高可用
