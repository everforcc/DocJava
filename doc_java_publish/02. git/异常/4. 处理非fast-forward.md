### 看这个提示是“非 fast-forward”被拒绝：远端同名分支有你本地没有的提交。根据你的目标分两种处理：

- 你“要覆盖远端”（因为你刚做了历史重写清理大文件）：强推

#### 1.1. 确认分支与远端

~~~shell
     git status
     git branch --show-current
     git remote -v
~~~

#### 1.2. 同步远端引用并清理已删分支

~~~shell
     git fetch --all --prune
~~~

#### 1.3. 强制推送（推荐带保护的强推）

- 覆盖当前分支：

~~~shell
       git push gitee HEAD:refs/heads/v_1 --force-with-lease
~~~

- 或简单：

~~~shell
       git push gitee v_1 --force-with-lease
~~~

-

~~~shell
       git push gitee --force-with-lease --all
       git push gitee --force-with-lease --tags
~~~

#### 1.4. 若 gitee 开了保护分支，先在仓库设置临时关闭保护，再执行上面的强推；完成后再开启。

- 你“不想覆盖远端”，而是把远端变更合入本地：先拉取再推送
- 拉取并保持线性历史（更常用）

~~~shell 
  git fetch gitee
  git rebase gitee/v_1
  git fetch gitee     git rebase gitee/v_1
~~~

- 如遇冲突，按文件解决后：

~~~shell
 git add <冲突文件...>
  git rebase --continue
  git add <冲突文件...>     git rebase --continue
~~~

- 完成后：

~~~shell
  git push gitee v_1
  git push gitee v_1
~~~  

2) 或者直接合并（会产生 merge commit）

~~~shell
   git pull gitee v_1
   git push gitee v_1
   git pull gitee v_1     git push gitee v_1
~~~

- 额外提示
- 你刚用历史重写移除了 .jar，大概率是要“覆盖远端”，请用 --force-with-lease。如果仍遇拒绝，多半是保护分支或权限问题。
- 覆盖完成后，再推 GitHub（一样先 git fetch --all --prune，再 git push origin --all/--tags）。若 GitHub 也开了保护分支，需要在设置里允许强推或改走普通推送。
- 如果远端分支名不同，替换上面命令里的 v_1 为实际分支名。
